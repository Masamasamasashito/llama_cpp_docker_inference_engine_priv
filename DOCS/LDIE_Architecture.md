# LDIE アーキテクチャ — 2つの運用パターン

LDIEには2つの運用パターンがあります。用途・予算・求める精度に応じて選択してください。

---

## Pattern A: 全ローカルLLM（電気代のみ）

```
┌─────────────────────────────────────────────────────────┐
│  Ubuntu 24.04 (OpenClaw)                                │
│                                                         │
│  ┌───────────────────┐                                  │
│  │ OpenClaw Agent     │  司令塔 + 作業役 = すべてローカル │
│  │ model: ldie/gemma  │                                  │
│  └────────┬──────────┘                                  │
│           │                                              │
└───────────┼──────────────────────────────────────────────┘
            │ API (LAN内)
            ▼
┌─────────────────────────────────────────────────────────┐
│  Windows 11 Pro (LDIE)                                  │
│                                                         │
│  ┌──────────────┐                                       │
│  │ Gemma 3 27B  │  :8081                                │
│  └──────────────┘                                       │
│  RTX 5090 (32GB VRAM)                                   │
└─────────────────────────────────────────────────────────┘
```

### 特徴

| 項目 | 内容 |
|---|---|
| 月額コスト | **電気代のみ（$25-50）** |
| インターネット依存 | **なし（完全オフライン運用可能）** |
| 精度 | 中〜高（27-32Bクラス） |
| トークン制限 | **無制限** |
| プライバシー | **最高（データが外部に一切出ない）** |
| 最適な用途 | 定型作業・ファイル操作・日常的な質問応答・プライバシー重視 |

### 推奨モデル

| 用途 | モデル | ポート |
|---|---|---|
| 汎用（安全性最高） | Gemma 3 27B | 8081 |
| 日本語重視 | Qwen3.5-27B | 8081 |
| コーディング | Qwen3-Coder-30B | 8081 |

> Pattern Aでは1モデルで司令塔と作業役を兼任します。

### openclaw.json

```jsonc
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "ldie/gemma-3-27b-it-Q4_K_M"
      }
    }
  },
  "models": {
    "providers": {
      "ldie": {
        "baseUrl": "http://192.168.1.100:8081/v1",
        "apiKey": "sk-local-your-secret-key-here",
        "api": "openai-completions",
        "models": [
          {
            "id": "gemma-3-27b-it-Q4_K_M",
            "name": "Gemma 3 27B (Local)",
            "reasoning": false,
            "input": ["text"],
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
            "contextWindow": 8192,
            "maxTokens": 4096
          }
        ]
      }
    }
  }
}
```

## Pattern B: 司令塔クラウド + 作業役ローカル（黄金バランス）

```
┌───────────────────────────────────────────────────────────────┐
│  クラウド（API課金: 月$20-80）                                  │
│                                                               │
│  ┌─────────────────────┐                                      │
│  │ 司令塔 Agent         │  Claude Opus / GPT-4o               │
│  │ (Orchestrator)      │  判断・計画・品質チェック              │
│  └──────┬──────────────┘                                      │
│         │ 指示（少量トークン）                                  │
└─────────┼─────────────────────────────────────────────────────┘
          │
          ▼
┌───────────────────────────────────────────────────────────────┐
│  Ubuntu 24.04 (OpenClaw)                                      │
│                                                               │
│  ┌───────────────────┐                                        │
│  │ OpenClaw Agent     │  司令塔の指示を受けてローカルLLMで作業  │
│  └────────┬──────────┘                                        │
│           │                                                    │
└───────────┼────────────────────────────────────────────────────┘
            │ API (LAN内)
            ▼
┌───────────────────────────────────────────────────────────────┐
│  Windows 11 Pro (LDIE)                                        │
│                                                               │
│  ┌──────────────┐ ┌──────────────┐                            │
│  │ Gemma 3 27B  │ │ Qwen3 Coder  │                            │
│  │ :8081        │ │ :8085        │                            │
│  │ 汎用作業     │ │ コード生成    │                            │
│  └──────────────┘ └──────────────┘                            │
│  RTX 5090 (32GB VRAM)                                         │
└───────────────────────────────────────────────────────────────┘
```

### 特徴

| 項目 | 内容 |
|---|---|
| 月額コスト | **クラウドAPI $20-80 + 電気代 $25-50** |
| インターネット依存 | **あり（司令塔がクラウド）** |
| 精度 | **最高（フロンティアモデルが判断・レビュー）** |
| トークン制限 | 司令塔は課金制、作業役は無制限 |
| プライバシー | 中（司令塔経由で一部データがクラウドに送信される） |
| 最適な用途 | 複雑なタスク・高品質なコード生成・エージェント自走 |

### なぜコスパが良いか

トークン消費の**95%以上は作業役で発生**します。

| 役割 | トークン量 | モデル | コスト |
|---|---|---|---|
| 司令塔（判断・計画・レビュー） | 全体の5% | Claude Opus / GPT-4o | API課金（少額） |
| 作業役（コード生成・ファイル操作・検索・繰り返し） | 全体の95% | LDIE（Gemma/Qwen） | **電気代のみ** |

### コスト比較

| 構成 | 月額 | 精度 |
|---|---|---|
| 全部クラウド（Claude Opus） | $500-2000+ | 最高 |
| **Pattern B（司令塔クラウド + 作業役LDIE）** | **$45-130** | **高〜最高** |
| Pattern A（全ローカル） | $25-50 | 中〜高 |

### 推奨モデル構成

| 役割 | モデル | 場所 | ポート |
|---|---|---|---|
| 司令塔 | Claude Opus 4.6 / GPT-4o | クラウドAPI | — |
| 作業役（汎用） | Gemma 3 27B | LDIE | 8081 |
| 作業役（コード） | Qwen3-Coder-30B | LDIE | 8085 |

> RTX 5090 (32GB) で2モデル同時運用が現実的な上限です。


## Pattern A vs B 選択ガイド

| 判断基準 | Pattern A（全ローカル） | Pattern B（ハイブリッド） |
|---|---|---|
| 予算 | 電気代のみ | +月$20-80のAPI課金が可能 |
| インターネット | 不要 / オフライン環境 | 必要 |
| プライバシー | 機密データを一切外部に出せない | 司令塔への送信は許容できる |
| タスク複雑度 | 定型作業・単純な指示 | 複雑な判断・マルチステップ |
| 品質要求 | 実用的な品質で十分 | フロンティアモデル級が必要 |

### 移行パス

Pattern A で運用を開始し、精度が不足する場面が増えたら Pattern B に移行できます。
変更は `openclaw.json` の `primary` モデルとクラウドプロバイダの追加のみです。
